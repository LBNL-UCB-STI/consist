Step Contracts¶

Step contracts expose resolved @define_step metadata without executing the step. Use them when an external orchestrator, schema builder, or validation layer needs the same declarative metadata that Tracker.run(...) and ScenarioContext.run(...) would use at execution time.

API reference¶

consist.core.step_contracts.StepContract dataclass ¶

Resolved metadata contract for a @define_step function.

This is the planning/introspection counterpart to the run invocation metadata used by Tracker.run and ScenarioContext.run. It intentionally carries the declarative fields that orchestrators need before execution without owning execution, output registration, or domain-specific coupler policy.

Attributes:

Resolved step run name after explicit overrides, decorator name_template, and default name templates are applied.

Resolved model name that execution would persist on the run record. When no explicit model is provided, this falls back to name.

Resolved human-readable step description.

Resolved configuration payload that would contribute to run identity.

Resolved config adapter, when decorator metadata or an explicit override provides one.

Resolved hash-only identity inputs. These are paths or labeled paths that affect run identity without being logged as artifacts.

consist.core.step_contracts.resolve_step_contract(step, *, name=None, model=None, description=None, config=None, adapter=None, identity_inputs=None, inputs=None, input_keys=None, optional_input_keys=None, tags=None, facet=None, facet_from=None, facet_schema_version=None, facet_index=None, year=None, iteration=None, phase=None, stage=None, outputs=None, output_paths=None, output_sets=None, cache_options=None, execution_options=None, default_name_template=None, allow_template=True, apply_step_defaults=True, consist_settings=None, consist_workspace=None, consist_state=None, runtime_kwargs=None, missing_name_error='resolve_step_contract requires a step name.') ¶

Resolve a decorated step's declarative metadata without executing it.

Explicit arguments use the same precedence as run execution: call-site values override decorator metadata, while callable decorator metadata is resolved with a StepContext populated from the supplied workflow context.

Raises:

Type Description

If runtime kwargs are provided both directly and through execution_options.runtime_kwargs.

Examples:

>>> import consist
>>> @consist.define_step(
...     outputs=lambda ctx: [f"results_{ctx.year}"],
...     name_template="{func_name}__y{year}",
... )
... def simulate():
...     pass
>>> contract = consist.resolve_step_contract(simulate, year=2030)
>>> contract.name
'simulate__y2030'
>>> contract.outputs
['results_2030']

consist.core.step_contracts.collect_step_contracts(steps, **context) ¶

Resolve a sequence of decorated steps with shared workflow context.

Parameters:

Name Type Description Default
steps iterable of Callable[..., Any]

Step functions to inspect.

 required
**context Any

Shared keyword arguments forwarded to resolve_step_contract for each step, such as year, iteration, runtime_kwargs, cache_options, or default_name_template.

 {}

Returns:

Type Description
list[StepContract]

Resolved contracts in the same order as steps.

Examples:

>>> import consist
>>> @consist.define_step(outputs=lambda ctx: [f"a_{ctx.year}"])
... def first():
...     pass
>>> contracts = consist.collect_step_contracts([first], year=2040)
>>> contracts[0].outputs
['a_2040']
